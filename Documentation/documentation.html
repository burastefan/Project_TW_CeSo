1. Introduction

1.1 Purpose

The purpose of this document is to present a detailed description of the web application CoSe (Crisis Containment Service). It will explain the purpose and features of the software, the interfaces of the software, what the software will do and the constraints under which it must operate. This document is intended for the potential users of the software.

1.2 Intended Audience and Reading Suggestions

The users of the applications are divided in three categories:

  • Local authorities specialized in crisis situations management
  • Civilians who can be affected by crisis situations
  • Unauthenticated users who have limited access to the application features

1.3 Product Scope

The purpose of the application is to develop a platform for the management of crisis situations (e.g. earthquakes, fires, floods, etc.) by competent authorities. The system will also provide useful information to the affected population regarding possible shelters and/or rescue routes, based on the category of the disaster.

2. Overall Description

2.1 Product Functions

  • Both the authorities and the civilians will have access to the following features:
    • Authentication:
      • Register
      • Register Validation Code
      • Login
      • Change password
    • Use location to see the events near them.
    • Search location manually on the map to see the events near it.
    • Filter events
    • Receive alerts on e-mail.
    • See events on map
    • See event history.
    • Export information about events (local + external) in csv format.
    • See shelters/rescue routes.
  • Only the authorities will have access to the following features:
    • Add crisis situation.
    • Review a crisis situation added by a civilian user.
    • Edit events.
    • Delete false/inappropriate events
    • Add shelter locations or rescue routes.
    • Send special instructions to the affected population.
  • Only the civilians will have access to the following features:
    • Add crisis situation which will be reviewed by the local authorities.
  • The user role (local authority/civilian) will be given by an administrator of the application.
  • Unauthenticated users will also be able to see crisis situations and view shelter & rescue routes.
  • Information about crisis events will be provided by two main sources:
    • APIs such as USGS Earthquake Catalog, NOAA National Weather Service and Environment Agency Real Time Flood-Monitoring.
    • Authorities and also citizens will be able to manually add an event (citizens will need approval from authorities) and provide further details about it.

2.2 User Classes and Characteristics

  • Civilian User: adds crisis situations that are going to be reviewed by authorities, views information about the crisis situations in his area (gravity, type of event, shelter, safety routes, etc.), views event history, reviews authorities based on how they handled the situation
  • Authority User: adds/removes crisis situations, provides useful information to civilians about these events, reviews civilian's crisis situations, views event history
  • Admin : gives roles to users, manages different aspects of the web application

2.3 Operating Environment

The developed product can be used on computer devices with a stable connection to the Internet and an installed browser.

    4. System Features

    • Notification Bar
      • Search Bar
        • User will be able to manually search events and see the results in the table.
      • Home Page
        • User will be able to see all events in a table and also filter them (category, level of danger, location, date of occurrence, status, etc.). User will be able to export all data from the table in a CSV format.
      • Map page
        • User will be able to see the ongoing events, possible shelters, safe routes based on live location or manually searched location.
      • Add Event Page
        • User will be able to manually search events and see the results in the table.
        • Citizen
          • User will be able to add a new event which will be reviewed by a local authority.
          User will be able to see the status of added events.
          User will be able to see event history.
        • Authority
          • User will be able to add a new event.
          User will be able to review events submitted by citizens.
          User will be able to see event history.
          User will be able to add shelters/safe routes.
      • Add Shelter Page
        • User will be able to add a new shelter.
      • Fires Page
        • This page will contain relevant information about fires.
      • Earthquakes page
        • This page will contain relevant information about earthquakes.
      • Floods page
        • This page will contain relevant information about floods.
      • Storms page
        • This page will contain relevant information about storms.
    • Side Panel
      • Register page
        • Guests can create an account and then use it to login into the application and access all the features provided by their account type (citizen/authority/admin).
      • Register Validate page
        • Guests can create an account and then use it to login into the application and access all the features provided by their account type (citizen/authority/admin).
      • Login page
        • Users of the app can login by entering their credentials and therefore access other functionalities of the app.
      • Change Password page
        • Users who have forgotten their password can unlock, retrieve, or reset it, by recieving a verification email.
